Standards de code et processus de Pull Request.

Workflow Git

Nous utilisons le modèle "Feature Branch".

git checkout -b feature/ma-nouvelle-fonc
# ... codage ...
git commit -m "feat: ajouter export CSV"
git push origin feature/ma-nouvelle-fonc

Standards de Code (Dart/Flutter)

  • Linting : Le code doit passer flutter analyze sans erreur.
  • Formatage : Utilisez dart format . avant chaque commit.
  • Architecture : Respectez le dossier viewmodels/ pour toute logique.
