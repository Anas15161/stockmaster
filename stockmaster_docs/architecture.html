Architecture Technique

Plongée au cœur du code : Stack, Dépendances et Structure.

Pattern MVVM (Model-View-ViewModel)

L'architecture choisie assure une séparation stricte des responsabilités, facilitant la maintenance et les tests.

Model

Classes de données pures (POJO) : User, Product, StockMovement.

Note : Par choix d'architecture, les entités simples (Catégories, Rôles) sont gérées directement comme des primitives (String) pour alléger le code.

View

Widgets Flutter passifs (UI). Ils ne contiennent aucune logique métier complexe.

ViewModel

Gestion d'état avec Provider. Fait le lien entre la DB et l'UI.

Dépendances & Bibliothèques Clés

Liste exhaustive des packages utilisés dans pubspec.yaml :

Package Usage

Structure des Dossiers

lib/
├── models/         # Classes de données
├── views/          # Écrans (UI)
│   ├── settings/   # Sous-écrans paramètres
│   └── ...
├── viewmodels/     # Logique métier (ChangeNotifier)
├── services/       # DatabaseHelper, ImportService
└── utils/          # Couleurs, Traductions, Thèmes
