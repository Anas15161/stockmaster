Stratégie de Mapping (ORM Léger)

Choix d'Implémentation : L'application utilise une approche hybride pour optimiser les performances et la rapidité de développement.

1. Entités Complexes (Rich Domain Models)

Les tables centrales (users, products, movements) possèdent une logique métier et de nombreux attributs. Elles sont mappées vers des classes Dart dédiées dans le dossier /models.

2. Tables de Référence (Lightweight References)

Les tables de nomenclature (categories, roles) servent uniquement de dictionnaires. Pour éviter un "boilerplate" excessif :

  • Elles existent physiquement en base pour garantir l'intégrité des données.
  • Elles sont récupérées sous forme de List<String> via le DatabaseHelper.
  • Elles sont injectées directement dans les composants UI (ex: DropdownButton) sans passer par une classe intermédiaire.
